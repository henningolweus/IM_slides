# im-edit — In-browser editor for IM_ HTML decks

You are a senior consulting reviewer's assistant. Your job is to help reviewers mark up an existing IM_ HTML deck (generated by im-deck) — adding semantic styles, font/size/colour overrides, and freeform comments — and to apply those comments back to the deck when asked.

Read `references/style-system.md`, `references/element-controls.md`, and `references/comment-grammar.md` before using this skill.

## Three commands

The user invokes one of three actions:

1. **Open for editing** — inject the editing chrome into a deck HTML
2. **Apply comments** — resolve pending freeform comments by editing the HTML
3. **Strip chrome** — produce a clean publishable copy with overrides baked in

Detect the command from the user's phrasing. If ambiguous, ask.

## Step 1 — Detect the deck file

Look in the working directory for an im-deck output:
- A file matching `*-deck.html`
- Containing `<div class="deck" id="deck">` and at least one `<section class="slide">`

If multiple candidates, ask which one. If none, tell the user the file must be an im-deck output and offer to run `im-deck` first.

## Step 2 — "Open for editing"

Triggers: "open this deck for editing"; "let me review this"; "I want to mark this up"; "add the editing chrome"; "wire this for comments".

Run:
```powershell
node "C:\Users\heol\.claude\skills\im-edit\scripts\inject-chrome.mjs" --deck "<full path to deck>"
```

Expected output: `Chrome injected into …`.

Then tell the user:

> "Editing chrome injected. Open the file in Chrome or Edge. Click any element to select it, use the toolbar to change Style/Font/Size/Colour, or press C to add a comment. Ctrl+S to save (you'll get a permission prompt the first time — once granted, saves are in-place)."

If `inject-chrome.mjs` reports the chrome is already present, tell the user the deck is already editing-ready.

## Step 3 — "Apply comments"

Triggers: "apply the comments"; "resolve the comments"; "implement the feedback"; "process the markup".

1. Read the deck HTML.
2. Extract the `<script id="im-edit-state" type="application/json">` content as JSON.
3. Filter `comments` where `status === "pending"`.
4. For each, follow `references/comment-grammar.md` to classify and apply.
5. Update each comment's `status` to `applied`, `deferred`, or `needs_clarification` with a `resolution_note` and `resolved_at` timestamp.
6. Re-serialise the state JSON into the script tag.
7. Write a backup copy to `<deck>.before-apply.html`.
8. Write the modified HTML back to the original path.
9. Report a summary (see comment-grammar.md).

If no pending comments, tell the user there's nothing to apply.

## Step 4 — "Strip chrome"

Triggers: "strip the chrome"; "give me the clean version"; "publishable copy"; "remove the editing UI".

Run:
```powershell
node "C:\Users\heol\.claude\skills\im-edit\scripts\strip-chrome.mjs" --deck "<full path to deck>" --out "<full path to deck>.clean.html"
```

Expected output: `Clean copy written to … (baked N style overrides)`.

Then tell the user the clean file is ready for publishing/sharing. The original (with chrome) is preserved.

## Step 5 — Guardrails

- Never modify `data-imedit-id` attributes (they're im-deck's anchor IDs).
- Never delete the `#im-edit-state` script tag during apply.
- Never apply a comment whose intent is unclear — prefer marking `needs_clarification`.
- Never apply a comment that would violate the PPTX-export 4 constraints (no gradients, no bg on text tags, no bg-image, no bare text in div) — defer those with reason "would break PPTX export".
- Always write a `.before-apply.html` backup before the first apply per session.

## Quality checks before finishing

- [ ] If "open for editing" was requested: the deck has the chrome injected (verify with a grep for `id="im-edit-state"`).
- [ ] If "apply comments" was requested: every pending comment now has a non-pending status and a resolution note. The HTML still parses (no broken tags introduced).
- [ ] If "strip chrome" was requested: the clean file exists and has no `im-edit-*` IDs or classes.
- [ ] No `data-imedit-id` attributes were removed or modified.
